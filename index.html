📐 Scoring-Methodik

Drei-Säulen-System nach CHAOSS-Metriken-Modellen. Benchmarks aus den Top-100 Open-Source-Projekten auf GitHub (täglich via GitHub Action aktualisiert). Jede Kennzahl wird per Ratio-Normalisierung gegen den Median der Top-100 verglichen: Score = min(100, Wert / Median × 100).

⚡ Aktivität & Community 40%

Commit-Häufigkeit (50%), Tage seit letztem Commit (30%), Anzahl Contributors (20%). Direktester Indikator für Projekt-Gesundheit — aktiver Code bedeutet aktive Entwicklung.

🔧 Reaktionsfähigkeit & Wartung 35%

Issue-Schließungsrate (25%), Release-Häufigkeit (25%), Ø Schließzeit (25%), Issue-Engagement (15%), Fehlerbehebungs-Produktivität (10%). Misst wie professionell das Projekt gewartet wird.

📡 Reichweite & Dokumentation 25%

Fork-Ratio (35%), Dokumentation README+CONTRIBUTING (35%), Stars/Alter log-skaliert (30%). Zeigt Verbreitung und Zugänglichkeit des Projekts.
